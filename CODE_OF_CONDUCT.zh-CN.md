# crop 行为准则

语言：[English](CODE_OF_CONDUCT.md) | [한국어](CODE_OF_CONDUCT.ko.md) | 简体中文 | [日本語](CODE_OF_CONDUCT.ja.md)

本文是简体中文译本，仅为方便阅读提供。如不同语言版本含义不一致，以 English 版本为准。

`crop` 仓库希望成为一个小而实用、相互尊重的开源协作空间。本行为准则适用于与本仓库相关的所有公开互动，包括 Issues、Pull Requests、Discussions、code review、文档和发布准备讨论。

## 期望行为

- 尊重他人，并假定对方有善意。
- 用复现步骤、证据和 tradeoff 支撑观点。
- 将批评集中在代码、文档、政策和用户影响上，而不是个人身上。
- 尊重他人的时间和上下文，并保持请求清晰。
- 谨慎处理 security、privacy、license 和 branding 主题，并优先使用可验证的依据。
- 发现错误时，提出可执行的修正路径。

## 不可接受的行为

- 侮辱、骚扰、威胁或恐吓
- 歧视性语言或排他性玩笑
- 不适当的性或暴力内容
- 未经同意公开个人信息、私人联系方式或敏感安全信息
- 反复离题争论、spam 或故意干扰
- 向 maintainer 或 contributor 施压，要求不合理的响应时间、决定或实现

## 报告和处理

如果你发现违反行为准则的情况，请通知 maintainer。

- 如果可以公开处理，可以在 GitHub Issue 或 Discussion 中简要报告。
- 如果包含个人信息、安全问题或敏感安全细节，不要公开发布细节。
- 如果需要私下协调，请通过 GitHub profile 中显示的联系方式联系 maintainer `@postmelee`，或只留下最小公开说明，例如“需要私下跟进的行为准则报告”。

Maintainer 可能采取以下措施：

- 要求参与者重新聚焦讨论
- 要求编辑或隐藏评论
- 锁定 Issue、Pull Request 或 Discussion
- 限制重复违规者参与
- 通过 GitHub 平台报告工具进一步处理

在可行范围内，信息只会分享给需要知情的人。已经公开的评论和活动记录只能在 GitHub 平台政策和权限模型范围内调整。

## 适用范围和限制

本文定义最低协作期望。它不是法律建议、雇佣政策，也不能替代平台规则。GitHub Terms、GitHub Community Guidelines、仓库许可证和安全政策也可能适用。
