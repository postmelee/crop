# Task #39 수행계획서 - Preview 모달 backdrop dismiss와 inline padding 정렬

GitHub Issue: [#39](https://github.com/postmelee/crop/issues/39)
마일스톤: M020

## 목적

visible viewport와 full page capture preview 모달의 닫힘 UX와 레이아웃을 보정한다. 모달 밖 어두운 backdrop을 클릭하면 `Esc`/Cancel과 같은 cleanup 경로로 overlay 전체가 닫히고, 모달 내부 클릭과 Copy/Save/Retry 동작은 영향을 받지 않아야 한다.

또한 preview 모달이 화면을 과도하게 차지하지 않도록 크기 상한과 표시 폭을 조정하고, preview 이미지 좌우 edge와 상단 toolbar Save 버튼의 우측 edge가 같은 inline padding 기준으로 정렬되게 한다.

## 배경

#15에서 visible/full page capture preview dialog와 상단 toolbar 기반 Copy/Save 흐름을 구현했고, 품질 기준 P6-29b/P6-29c/P6-35/P6-36으로 preview 위치, 정렬, shortcut, flicker 방지를 고정했다. 현재 화면에서는 preview dialog가 넓은 viewport 대부분을 차지해 backdrop click 영역이 좁고, 모달 밖 어두운 부분을 클릭해도 닫히지 않는다.

작업지시자는 모달 밖 어두운 부분 클릭을 dismiss로 처리하는 편이 더 좋은 UX라고 판단했고, 추가로 모달 좌우 padding을 상단 Save 버튼의 우측 padding 기준과 맞추되 backdrop click과 충돌하지 않도록 모달이 화면을 너무 많이 차지하지 않게 해 달라고 요청했다.

## 설계 방향

- backdrop dismiss는 `.crop-preview` container 자체를 클릭한 경우에만 처리한다. composed path에 `.crop-preview-dialog` 내부 요소가 있으면 dismiss하지 않는다.
- dismiss는 새 cleanup 경로를 만들지 않고 기존 `requestClose()`를 사용해 `Esc`/Cancel과 같은 상태 전이, event listener 제거, host 제거 흐름을 공유한다.
- `pendingCapture` 또는 preview Copy/Save 처리 중에는 backdrop dismiss를 무시해 action 완료와 dismiss가 중복으로 overlay를 제거하지 않게 한다.
- `pointerdown`과 `click` 이벤트의 기존 selected outside reset, drag selection, preview wheel handling을 깨지 않도록 preview 상태 우선 분기를 둔다.
- CSS는 기존 `--crop-preview-inline-padding` 변수를 기준으로 footer와 surface의 inline padding을 공유한다. 필요하면 dialog max width/height와 viewport padding을 함께 줄여 backdrop click 영역이 항상 남도록 한다.
- preview 이미지가 height-fit 때문에 작아지는 경우에도 toolbar Save edge와 image edge가 같은 기준을 사용하도록 surface/image wrapper 정렬 방식을 검토한다.
- 모바일/좁은 viewport media query는 기존 preview 여백 축소 정책을 유지하되, backdrop click target과 버튼 접근성을 우선한다.

## 리스크

- **event sequence 충돌**: `pointerdown`/`click` 분기 순서가 selected outside reset, drag selection, preview action click과 충돌할 수 있다. preview 상태 우선 조건과 회귀 테스트로 고정한다.
- **pending action 중복 cleanup**: Copy/Save 처리 중 backdrop click이 들어오면 overlay 제거와 action 완료가 겹칠 수 있다. `pendingCapture` guard를 명시한다.
- **레이아웃 회귀**: dialog 크기를 줄이면 full page preview 가독성이나 visible preview no-scroll 기준이 흔들릴 수 있다. 기존 P6-29b/P6-29c 테스트를 유지하면서 수치를 보정한다.
- **모바일 여백 부족**: 좁은 viewport에서는 backdrop target과 이미지 표시 면적 사이 tradeoff가 있다. 모바일 media query는 버튼 접근성과 최소 backdrop target을 우선한다.

## 승인 요청 사항

- preview backdrop 직접 클릭을 `Esc`/Cancel과 같은 overlay cleanup으로 처리하는 방향을 승인한다.
- preview dialog 내부 클릭, scroll, Copy/Save/Retry/Cancel은 dismiss와 충돌하지 않게 guard를 두는 방향을 승인한다.
- dialog가 화면을 과도하게 차지하지 않도록 최대 크기와 viewport 여백을 보정하는 방향을 승인한다.
- Save 버튼 우측 edge와 preview image edge가 같은 inline padding 기준을 쓰도록 CSS를 조정하는 방향을 승인한다.
- 품질 기준은 `mydocs/tech/task_m020_8_quality_matrix.md`에 최소 갱신하고 README 계열은 수정하지 않는 방향을 승인한다.
