# Task #12 수행계획서

GitHub Issue: [#12](https://github.com/postmelee/crop/issues/12)
마일스톤: M020

## 목적

드래그 선택 중 pointer가 viewport 가장자리 근처에 머무를 때 페이지가 자동으로 스크롤되도록 구현한다. 선택 영역은 기존처럼 page/document 좌표 기준으로 유지되어야 하며, 스크롤 중에도 dashed selection과 mask가 사용자가 선택 중인 문서 영역을 따라 자연스럽게 갱신되어야 한다.

이번 task는 #8 Phase 6 품질 검증 뒤 남은 M020 안정화 후속 작업 중 하나다. full page capture나 scroll stitching을 구현하는 작업이 아니라, 현재 visible viewport 기반 MVP에서 드래그 선택 UX만 Firefox Screenshots에 더 가깝게 보정하는 작업이다.

## 배경

#5에서 page-coordinate hover/click/drag selection과 scroll-follow highlight를 구현했고, #8에서 실제 smoke를 통해 Copy/Save, zoom, viewport 밖 요소 제한을 정리했다. 그러나 현재 `crop`은 드래그 중 pointer가 화면 가장자리로 이동해도 문서가 자동으로 스크롤되지 않는다. 따라서 화면 밖에 있는 문서 영역까지 한 번의 드래그로 선택하기 어렵다.

Issue #12는 Firefox Screenshots 원본의 `SCROLL_BY_EDGE`, `scrollIfByEdge()`, `scrollWindow()` 흐름을 참고해 이 UX 차이를 줄이기 위해 등록됐다. Chrome MV3에서는 Firefox privileged actor나 debugger 권한 없이 일반 content script에서 접근 가능한 `window.scrollBy()`/`scrollTo()`와 현재 overlay state 구조를 사용해야 한다.

## 범위

### 포함

- `src/content/overlay/`의 drag selection pointer move 흐름에 edge auto-scroll 연결
- viewport 상/하/좌/우 edge 근처 pointer 위치 감지
- requestAnimationFrame 또는 timer 기반 반복 스크롤 loop 설계
- 스크롤 중 page-coordinate drag end point와 selected rectangle 갱신
- vertical/horizontal scroll 동작
- Escape/Cancel, pointerup, overlay 제거 시 scroll loop 정리
- 기존 click selection, 일반 drag selection, selected outside reset, Copy/Save capture 동작 회귀 방지
- 단위 테스트와 가능 범위의 browser/CDP smoke 또는 작업지시자 수동 smoke 지침

### 제외

- full page capture
- scroll stitching
- resize/move handles 기반 edge auto-scroll
- iframe 내부 문서 스크롤
- cross-origin iframe 내부 선택 고도화
- selected-state size badge와 action button parity
- `debugger` 권한 추가
- `<all_urls>` host 권한 추가

## 설계 방향

- 기존 overlay state는 `dragStart`와 `selectedRect`를 page 좌표로 저장한다. edge auto-scroll도 이 좌표 체계를 유지하고, render 직전에만 viewport 좌표로 투영한다.
- edge 감지와 scroll delta 계산은 가능하면 작은 순수 helper로 분리해 단위 테스트를 붙인다.
- scroll loop는 dragging/draggingReady 상태에서만 살아 있어야 한다. pointerup, Escape, Cancel, overlay remove, capture 시작 시에는 반드시 중단한다.
- pointer가 edge 밖으로 나가거나 viewport 내부 안전 영역으로 돌아오면 자동 스크롤을 멈춘다.
- 스크롤이 발생한 frame에서는 마지막 pointer 위치와 최신 `scrollX`/`scrollY`를 조합해 `dragMove`를 재계산한다.
- Firefox 원본의 동작 감각을 참고하되 Chrome content script에서 안전한 API만 사용한다.
- MVP 권한 정책은 유지한다. `manifest.json`에 새 권한을 추가하지 않는다.

### 통합 검증

- drag selection 중 pointer가 viewport 상하좌우 edge 근처에 머무르면 페이지가 자동 스크롤된다.
- 자동 스크롤 중 selected rectangle이 page 좌표 기준으로 확장되고 화면에서 자연스럽게 갱신된다.
- pointer가 edge 영역을 벗어나거나 pointerup이 발생하면 자동 스크롤이 멈춘다.
- Escape/Cancel/overlay remove 시 scroll loop가 남지 않는다.
- click selection, 일반 drag selection, selected outside reset, Copy/Save가 회귀하지 않는다.
- `debugger`, `<all_urls>` 권한이 추가되지 않는다.

- `git diff --check`가 경고 없이 통과한다.

## 리스크

- **브라우저 event/scroll 타이밍 차이**: pointermove와 scroll event 순서가 환경별로 다를 수 있다. scroll loop는 최신 pointer와 최신 window dimensions를 매 frame 다시 읽도록 설계한다.
- **무한 scroll loop 위험**: pointerup, cancel, remove cleanup을 누락하면 overlay 제거 뒤에도 scroll이 계속될 수 있다. cleanup 경로를 테스트와 smoke로 확인한다.
- **선택 좌표 drift**: scroll 중 viewport 좌표와 page 좌표를 섞으면 선택 영역이 밀릴 수 있다. state에는 page 좌표만 저장한다는 원칙을 유지한다.
- **Chrome 자동화 한계**: 실제 확장 drag와 scroll은 자동화가 불안정할 수 있다. 자동 테스트로 helper/state를 고정하고, 필요한 경우 작업지시자 수동 smoke 지침을 제공한다.
- **범위 초과**: full page capture, scroll stitching, resize handles, iframe 내부 스크롤은 이번 task에 포함하지 않는다.
