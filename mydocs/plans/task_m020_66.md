# 수행계획서

GitHub Issue: [#66](https://github.com/postmelee/crop/issues/66)
마일스톤: M020

## 목적

viewport 밖으로 일부 나간 selected 영역을 Copy/Save할 때 `capturePageRectTiles()` 기반 stitching 경로가 광고 iframe 또는 lazy compositor surface를 검정 placeholder로 캡처하는 문제를 보정한다.

이번 task는 정상 동작 중인 일반 본문 캡처와 viewport 안 selected crop을 흔들지 않도록, selected page rect stitching 경로에 한정해 scroll planning과 제한적 retry를 개선한다.

## 배경

사용자 재현에서 같은 Adobe 광고 영역은 전체가 viewport 안에 들어온 상태로 선택하면 `600 x 1200` PNG가 정상 생성됐다. 반대로 선택 영역 일부가 viewport 밖에 있는 상태에서 실행하면 결과가 같은 크기의 불투명 검정 PNG가 됐다.

분석 결과 선택 크기, DPR 변환, clipboard 전달은 정상이며, 차이는 `captureSelectedRegion()`이 viewport 완전 포함 여부에 따라 단일 visible crop 경로와 selected page rect stitching 경로로 분기하는 데 있다. 현재 `createPageRectTilePlan()`은 선택 영역이 한 viewport 안에 들어갈 수 있어도 tile 시작점을 selected rect top/left에 맞추므로, 이미 대부분 보이는 선택 영역도 불필요하게 크게 프로그램 스크롤될 수 있다.

기존 #26은 viewport 밖 selected capture가 visible viewport 교집합으로 잘리는 문제를 해결했고, #40은 scroll/capture 직후 Chrome paint artifact를 별도 분석했다. 이번 task는 두 작업의 범위를 유지하면서 selected stitching edge case만 좁게 보정한다.

## 범위

### 포함

- selected page rect tile planning에서 한 viewport 안에 들어가는 선택 영역의 scroll 위치 계산 검토와 보정
- 현재 scroll 위치 기준 최소 이동으로 선택 rect 전체를 viewport 안에 넣는 정책 적용
- selected stitching 경로에 한정한 검정 placeholder 감지와 짧은 bounded retry 정책 검토
- `full-page-capture` 및 관련 stitching/crop 테스트 보강
- manual smoke 절차와 잔여 리스크를 작업 산출물에 기록

### 제외

- `debugger`, `<all_urls>`, broad host permission 추가
- full page capture 전체 알고리즘 교체
- viewport 안 selected crop의 단일 visible capture 경로 변경
- cross-origin iframe 내부 DOM 접근 지원
- 광고 iframe 자체의 콘텐츠 로딩 제어
- 공식 사용자 문서 루트 생성 또는 사용자-facing 문서 갱신

## 설계 방향

- 우선순위는 scroll planning 보정이다. 선택 rect가 viewport보다 작거나 같으면 current scroll을 기준으로 rect 전체가 보이는 scroll 범위에 가장 가까운 값을 선택한다.
- 선택 rect가 viewport보다 큰 multi-tile case는 기존 segment 기반 tiling을 유지해 일반 본문과 full page capture의 동작 차이를 최소화한다.
- retry는 selected stitching 경로에서만 사용한다. raw tile capture가 all-black 또는 극단적 저분산으로 의심되는 경우 짧은 지연 후 제한 횟수만 재시도하는 방향을 검토한다.
- full page capture와 visible viewport capture는 변경하지 않는다. 공용 helper를 바꾸더라도 호출 옵션으로 selected page rect에만 새 정책이 적용되게 한다.
- 권한 경계는 현재 manifest 범위를 유지한다.

## 잠정 단계

- **Stage 1 — scroll planning 계약 고정**
  - selected page rect가 viewport보다 작거나 같은 경우의 최소 이동 scroll 정책을 테스트로 정의한다.
  - 기존 multi-tile, clamp, scroll restoration 테스트가 유지되는지 확인한다.
- **Stage 2 — selected stitching runtime 보정**
  - selected page rect capture에 최소 이동 scroll 정책을 연결한다.
  - selected 경로에 한정한 검정 placeholder 감지와 bounded retry 적용 여부를 구현계획서 기준으로 결정하고 적용한다.
- **Stage 3 — 회귀 검증과 수동 smoke 기준 정리**
  - focused test, typecheck, 권한 grep을 실행한다.
  - 광고 iframe 재현 시나리오와 일반 본문 offscreen capture 확인 절차를 단계 보고서에 기록한다.
- **Stage 4 — 최종 보고와 PR 준비**
  - 최종 보고서를 작성하고 전체 검증 결과, 잔여 리스크, PR 범위를 정리한다.

### 통합 검증

- viewport 안 selected crop은 기존 단일 visible capture 경로를 유지한다.
- viewport 밖 일반 본문 selected capture는 출력 크기와 내용이 회귀하지 않는다.
- viewport 밖 광고/iframe selected capture는 불필요한 top-align scroll을 피하고, 검정 placeholder 캡처 가능성이 줄어든다.
- `git status --short`가 PR 준비 전 빈 출력이다.
- `git diff --check`가 경고 없이 통과한다.

## 리스크

- **광고 iframe compositor 타이밍**: Chrome이 cross-origin iframe surface를 늦게 합성하면 최소 이동 scroll만으로 완전히 해결되지 않을 수 있다. bounded retry를 selected stitching에 한정해 보조책으로 검토한다.
- **검정 콘텐츠 오탐**: all-black 감지는 실제 검정 콘텐츠를 retry할 수 있다. retry 횟수와 적용 조건을 제한하고, 실패해도 기존 결과를 반환하는 정책을 검토한다.
- **공용 helper 영향**: `capturePageRectTiles()`는 full page capture와 공유된다. 새 정책은 selected page rect 옵션으로 좁히거나 기존 full page 호출에는 영향을 주지 않게 한다.
- **수동 재현 한계**: 실제 광고 소재와 iframe paint timing은 외부 서비스 상태에 좌우된다. 자동 테스트는 scroll planning과 retry 계약을 검증하고, 수동 smoke는 절차와 관찰 기준으로 보완한다.

## 승인 요청 사항

- selected page rect가 한 viewport 안에 들어갈 때 현재 scroll 기준 최소 이동 scroll을 쓰는 설계 방향 승인
- 검정 placeholder retry를 selected stitching 경로에 한정하고 bounded 정책으로 검토하는 방향 승인
- `debugger`, `<all_urls>`, broad host permission을 추가하지 않는 제외 범위 승인

승인되면 `task_m020_66_impl.md`에서 단계별 산출물, 검증 명령, 커밋 메시지를 구체화한다.
