# Task #14 수행계획서

GitHub Issue: [#14](https://github.com/postmelee/crop/issues/14)
마일스톤: M020

## 목적

Chrome MV3 권한 제약 안에서 iframe/nested context 요소 선택 지원 범위를 명확히 정하고 구현한다. 현재 crop은 iframe을 만나면 iframe element 자체를 fallback target으로 처리한다. 이번 task는 same-origin iframe 내부 요소는 가능한 범위에서 parent page 좌표로 변환해 highlight/selection할 수 있게 하고, cross-origin iframe은 안전한 fallback과 제한 문서화를 유지한다.

open shadow root hit-test는 #13 기준으로 이미 deepest open shadow element traversal이 동작하므로, 이번 task에서는 iframe 지원 변경이 기존 open shadow root 동작을 깨지 않는지 회귀 검증한다. `debugger`, `<all_urls>` 권한 없이 MVP 권한 정책을 유지한다.

## 배경

Firefox Screenshots는 privileged `browsingContext` actor를 통해 nested browsing context 내부 element rect를 조회할 수 있다. 반면 Chrome MV3 content script는 cross-origin iframe 내부 DOM에 접근할 수 없고, 접근 가능한 same-origin/srcdoc iframe과 접근 불가능한 cross-origin iframe을 명확히 분리해야 한다.

현재 `src/firefox-derived/overlay-helpers.ts`의 `getElementFromPoint()`는 top document와 open shadow root를 탐색하지만 iframe을 만나면 `{ unsupportedReason: "iframe" }`으로 반환한다. Phase 6 fixture에는 `srcdoc` same-document iframe과 open shadow host가 이미 있고, 품질 매트릭스의 P6-10/P6-24/P6-25가 이번 task의 기준점이다.

## 범위

### 포함

- same-origin/srcdoc iframe 내부 `elementFromPoint()` 탐색
- iframe 내부 element `getBoundingClientRect()`를 parent page 좌표로 변환하는 helper
- nested same-origin iframe이 가능할 경우 최소 1단계 이상 중첩 좌표 변환 검토와 테스트
- open shadow root traversal 회귀 테스트 유지
- cross-origin iframe 접근 실패 시 iframe element fallback 유지
- fallback reason 또는 제한 상태를 테스트 가능한 형태로 정리
- Phase 6 fixture 또는 신규 fixture에 same-origin iframe 내부 target smoke 보강
- README와 품질 매트릭스의 iframe 제한/지원 범위 갱신
- `debugger`, `<all_urls>` 권한이 추가되지 않는지 검증

### 제외

- `debugger` 권한 추가
- `<all_urls>` host 권한 추가
- cross-origin iframe 내부 DOM 강제 접근
- closed shadow DOM 내부 선택
- Firefox privileged actor/browsingContext 포팅
- full page capture와 scroll stitching

## 설계 방향

- 기존 Firefox-derived helper boundary를 유지한다. iframe hit-test/rect 변환 알고리즘은 `src/firefox-derived/overlay-helpers.ts`에 두되, Chrome에서 사용할 수 없는 privileged API는 포팅하지 않는다.
- top document와 same-origin iframe document는 동일한 `HitTestResult` 계약으로 반환한다. result에는 실제 hit element와 parent viewport 기준 rect 또는 page 기준 rect 산출에 필요한 정보가 포함되어야 한다.
- iframe 내부 좌표 변환은 브라우저가 제공하는 `iframe.getBoundingClientRect()`와 iframe document 내부 element rect를 합산하는 방식으로 시작한다. iframe border, iframe document scroll, parent window scroll을 명시적으로 테스트한다.
- cross-origin iframe은 `contentDocument` 접근 실패 또는 null을 정상 제한으로 처리한다. 예외를 삼키되 iframe element fallback과 제한 사유를 유지한다.
- overlay runtime은 기존 `getBestRectForElement()` 호출 흐름을 가능한 유지한다. helper 단계에서 parent coordinate rect를 제공할 수 있으면 runtime 변경을 최소화한다.
- open shadow root와 iframe traversal이 섞인 경우에는 접근 가능한 open shadow 안의 iframe fallback 또는 same-origin iframe traversal을 명확히 테스트한다.
- 사용자-facing 브랜딩은 `crop`만 사용한다. Firefox/Mozilla 명칭은 upstream 출처와 기술 근거 맥락에서만 쓴다.

## 잠정 단계

- **Stage 1 — iframe hit-test 모델과 좌표 변환 기준 작성**
  - 현재 iframe fallback, open shadow traversal, Firefox 원본 iframe 처리 구조를 분석한다.
  - same-origin iframe 접근 가능 조건, cross-origin fallback 조건, rect coordinate contract를 확정한다.
  - helper 단위 테스트를 먼저 추가한다.
- **Stage 2 — same-origin iframe traversal 구현**
  - `getElementFromPoint()`가 same-origin/srcdoc iframe 내부 target을 찾도록 구현한다.
  - iframe 내부 rect를 parent viewport/page 좌표로 변환해 기존 `getBestRectForElement()` 흐름과 연결한다.
  - iframe document scroll, iframe border, parent scroll 조건을 테스트한다.
- **Stage 3 — cross-origin fallback과 nested/open-shadow 회귀 정리**
  - cross-origin iframe 접근 실패를 예외 없이 fallback으로 처리한다.
  - open shadow root traversal과 iframe fallback/지원 조합을 회귀 테스트한다.
  - 권한 추가가 없는지 manifest/README/테스트로 고정한다.
- **Stage 4 — fixture, smoke, 문서와 최종 보고**
  - Phase 6 fixture 또는 전용 fixture에 same-origin iframe 내부 target smoke를 추가한다.
  - CDP 또는 수동 smoke로 iframe 내부 hover/click selection, cross-origin fallback, open shadow 회귀를 확인한다.
  - README, 품질 매트릭스, 최종 보고서, 오늘할일을 갱신하고 PR 준비 상태로 정리한다.

## 리스크

- **브라우저 보안 경계**: cross-origin iframe 내부 DOM 접근은 Chrome MV3에서 제한된다. 접근 실패를 정상 경로로 처리하고, 권한 추가 없이 fallback으로 제한한다.
- **좌표 변환 오차**: iframe element rect, iframe document scroll, border, parent scroll을 섞으면 highlight가 밀릴 수 있다. viewport/page 좌표 contract를 helper 테스트로 먼저 고정한다.
- **nested context 복잡도**: nested iframe을 모두 지원하려 하면 범위가 커질 수 있다. Stage 1에서 지원 depth와 fallback 기준을 확정하고, 구현은 same-origin 가능한 범위로 제한한다.
- **기존 open shadow 회귀**: iframe traversal 추가가 open shadow root traversal을 깨뜨릴 수 있다. 기존 shadow 테스트를 유지하고 조합 테스트를 추가한다.
- **권한 범위 회귀**: `debugger`와 `<all_urls>`는 MVP 제약상 추가하지 않는다. manifest와 테스트/grep으로 확인한다.
- **자동화 한계**: 실제 cross-origin iframe smoke는 로컬 fixture만으로 완전히 대체하기 어렵다. 접근 불가를 재현하는 fixture 또는 대표 웹 수동 smoke 중 하나를 Stage 3에서 확정한다.
