# Task #40 수행계획서

GitHub Issue: [#40](https://github.com/postmelee/crop/issues/40)
마일스톤: M020

## 목적

전체 페이지 캡처 후 preview modal 내부를 빠르게 스크롤할 때 하단에 하얀 blank 띠가 순간 노출되는 시각 결함을 보정한다. 사용자는 캡처 결과를 검토하는 중이므로, preview 스크롤 중 새로 노출되는 영역이 흰색으로 번쩍이면 저장 PNG에 문제가 있는 것처럼 보인다.

이번 task는 preview 표시 품질과 실제 stitched PNG 결과를 분리해 판단한다. 문제가 preview `<img>` 렌더링 fallback이면 UI 표시만 고치고, stitched canvas에 실제 투명/빈 seam 가능성이 있으면 저장 결과까지 안전하게 보정한다.

## 배경

작업지시자가 2026-06-04에 시각 검증 영상을 공유했다. 영상 분석 결과 scroll이 집중된 구간은 약 `0.80s ~ 1.35s`였고, 원본 프레임 기준 `1.20s` 하단에 흰 blank 띠가 노출된 뒤 `1.25s`에는 실제 콘텐츠로 다시 채워졌다.

프레임 분해 결과 모달 외곽 rounded corner나 `.crop-preview-dialog` clipping 문제는 아니었다. 흰 영역은 preview 이미지 폭 전체를 가로지르는 새로 노출된 하단 영역에 나타났다. 현재 CSS는 `.crop-preview-image { background: #ffffff; }`를 사용하므로, 큰 PNG가 스크롤 중 한 프레임 늦게 raster/paint되거나 PNG에 투명 pixel이 있으면 흰색 fallback이 보일 수 있다.

선행 Task #35에서 full page oversized downscale fallback이 들어갔고, downscaled stitched output을 preview에서 스크롤하는 실제 사용 시나리오가 늘었다. #40은 그 후속 품질 보정이다.

## 범위

### 포함

- preview modal 스크롤 중 하얀 blank 띠 노출 원인 확정
- `.crop-preview-image`, `.crop-preview-surface`, `.crop-preview-dialog` 렌더링 정책 보정
- 필요 시 `stitchCapturedTiles()` canvas 초기 fill 또는 tile draw rounding 방어 보강
- full page preview scroll 시각 회귀 기준 추가
- Phase 6 품질 매트릭스에 preview scroll blank 항목 추가
- 영상 분석 근거와 수동 smoke 절차 기록

### 제외

- Chrome Web Store Dashboard 입력값 확정 작업(#37)
- full page capture 권한 구조 변경
- Chrome `debugger` API 기반 screenshot
- `<all_urls>` 또는 broad host permission 추가
- browser-native full-page screenshot API 전환
- lazy loading, animation, layout shift, sticky/fixed 변화의 pixel-perfect 해결
- Store screenshot 또는 promotional image 제작

## 설계 방향

- 먼저 preview UI artifact와 저장 PNG artifact를 구분한다.
- preview UI만의 문제라면 `.crop-preview-image`의 흰 fallback 배경을 제거하거나 modal surface와 일관된 어두운 배경으로 맞춘다.
- saved PNG 자체에 투명 seam 가능성이 있으면 `stitchCapturedTiles()`에서 canvas를 안전한 배경색으로 초기화하거나 destination rect rounding 보정을 추가한다.
- 기본 저장 결과를 임의 색으로 오염시키지 않는다. 페이지 배경 추론이 불확실하면 preview-only fix를 우선하고 저장 PNG 변경은 테스트 근거가 있을 때만 적용한다.
- visible viewport preview는 기존처럼 내부 스크롤 없이 맞춰지는 동작을 유지한다.
- full page/selected stitching, overlay 숨김, scroll restoration, Copy/Save action pipeline은 회귀하지 않도록 focused tests를 함께 실행한다.

## 잠정 단계

- **Stage 1 — preview scroll artifact 원인 고정**
  - 영상 분해 결과와 현재 preview DOM/CSS/stitching contract를 정리한다.
  - preview-only 문제인지 saved PNG seam 가능성인지 구현계획서에서 판단할 기준을 고정한다.

- **Stage 2 — preview 렌더링 보정**
  - preview image/surface 배경과 scrolling paint fallback을 보정한다.
  - visible viewport preview no-scroll 동작과 full page preview scroll 동작을 회귀 테스트로 고정한다.

- **Stage 3 — stitching seam 방어 필요성 판단 및 보정**
  - canvas 초기 fill 또는 destination rounding 보정이 필요한지 테스트로 판단한다.
  - 필요할 때만 `stitch-image` 보정을 적용하고 저장 PNG 오염이 없도록 검증한다.

- **Stage 4 — 품질 매트릭스와 최종 검증**
  - Phase 6 matrix에 preview scroll blank 항목과 수동 smoke 절차를 추가한다.
  - 전체 검증과 최종 보고서를 작성한다.

## 리스크

- **재현 민감도**: 현상은 빠른 scroll 중 한 프레임짜리 paint artifact로 보인다. 자동 테스트만으로 완전 재현이 어려울 수 있어 영상 분석과 수동 smoke 기준을 함께 둔다.
- **저장 PNG 오염**: canvas fill을 잘못 적용하면 투명 배경을 의도한 페이지 결과가 변할 수 있다. 저장 결과 변경은 seam 근거가 있을 때만 적용한다.
- **브라우저 compositor 차이**: Chrome 버전, DPR, GPU compositing 상태에 따라 artifact 빈도가 달라질 수 있다. CSS 보정은 fallback 색 노출을 줄이는 방향으로 둔다.
- **#37 작업과 범위 충돌**: 현재 #37은 Chrome Web Store Dashboard 문서 작업이다. #40은 별도 worktree와 `local/task40`에서 진행하고 #37 브랜치 산출물을 건드리지 않는다.
