# Task #31 수행계획서

GitHub Issue: [#31](https://github.com/postmelee/crop/issues/31)
마일스톤: M030

## 목적

Chrome Web Store 배포 준비 과정에서 사용자-facing README의 문체를 정리합니다. 이번 task의 1차 목적은 `README.ko.md`를 한국어 사용자 안내 문서에 맞는 존댓말 문체(`~입니다`, `~합니다`, `~하세요`)로 전환하는 것입니다.

동시에 `README.zh-CN.md`와 `README.ja.md`는 각 언어권 README에서 자연스러운 격식과 현재 문체가 맞는지 점검합니다. 의미 변경이 필요한 작업이 아니라 문체와 사용자 안내 품질을 정리하는 작업으로 제한합니다.

## 배경

Task #28에서 사용자용 README와 다국어 README 링크가 정리되었습니다. 후속 검토에서 `README.ko.md`에 `확장이다`, `로드한다`, `연다`처럼 평어체가 남아 있어 일반 사용자 안내 문서로는 다소 건조하고 덜 정중하게 읽히는 문제가 확인되었습니다.

중국어 README는 한국어의 존댓말과 같은 직접 대응 문법이 없어 중립 안내체가 일반적입니다. 일본어 README는 이미 `です/ます` 정중체 중심으로 작성되어 있어, 현재 문체의 일관성 유지 여부를 확인하는 쪽이 적절합니다.

## 범위

### 포함

- `README.ko.md`의 사용자-facing 문장을 존댓말 문체로 수정합니다.
- `README.zh-CN.md`의 문체가 중국어 README 관행상 자연스러운 중립 안내체인지 점검합니다.
- `README.ja.md`의 문체가 일본어 README 관행상 자연스러운 `です/ます` 정중체인지 점검합니다.
- 중일 README에 수정이 필요하지 않다고 판단할 경우, 그 근거를 구현계획서 또는 보고서에 기록합니다.

### 제외

- 영어 `README.md` 구조 변경
- 기능 설명, 권한 설명, 개인정보 설명의 의미 변경
- Chrome Web Store 스토어 등록용 이미지, 스크린샷, 소개 영상 제작
- 신규 기능 구현, manifest 권한 변경, 빌드 설정 변경
- 다국어 README 전체 재번역

## 설계 방향

- 한국어 README는 사용자를 직접 안내하는 문서로 보고 `~입니다`, `~합니다`, `~하세요` 중심으로 맞춥니다.
- 명령형 단계 안내는 `여세요`, `켜세요`, `선택하세요`, `클릭하세요`처럼 자연스러운 안내형 존댓말을 사용합니다.
- 표, 코드, 파일명, 제품명, 권한명, UI label은 의미 보존을 위해 원문 표기를 유지합니다.
- 중국어 README는 존댓말을 기계적으로 추가하지 않습니다. 필요하면 `请`는 요청성 문장에만 제한적으로 사용하고, 기본은 중립 안내체를 유지합니다.
- 일본어 README는 `です/ます` 정중체를 기준으로 유지하며, 평서형 또는 과도하게 딱딱한 문장이 섞여 있는 경우에만 최소 수정합니다.

- **Stage 2 — README 문체 수정**

  - 산출물: 최종 보고서, 오늘할일 완료 갱신

## 검증 계획

### 단계별 검증

- Stage 1
  - `git diff --check`
  - 구현계획서의 문체 기준, 변경 파일, 검증 기준 수동 검토
- Stage 2
  - `git diff --check`
  - `README.ko.md` 평어체 잔여 문장 수동 검토
  - `README.zh-CN.md`, `README.ja.md` 문체 일관성 수동 검토
- Stage 3
  - `git diff --check`
  - `git status --short`
  - 최종 보고서에 중일 README 문체 판단과 한국어 README 변경 요약이 기록됐는지 확인

### 통합 검증

- `README.ko.md`가 사용자 안내 문서로 자연스러운 존댓말 문체로 읽힙니다.
- `README.zh-CN.md`와 `README.ja.md`는 각 언어권 문체 판단이 기록되고, 불필요한 의미 변경이 없습니다.
- `git status --short`가 PR 준비 전 빈 출력입니다.
- `git diff --check`가 경고 없이 통과합니다.

## 리스크

- **의미 변경 위험**: 문체 수정 중 기능 설명이나 제한사항 의미가 달라질 수 있습니다. 문장별로 기존 의미를 유지하며, UI label과 권한명은 그대로 둡니다.
- **중국어 과잉 정중화 위험**: 한국어 존댓말을 중국어에 기계적으로 반영하면 부자연스러운 README가 될 수 있습니다. 중립 안내체를 기본으로 유지합니다.
- **일본어 불필요 수정 위험**: 이미 정중체인 문서를 과도하게 수정하면 오히려 번역 품질이 흔들릴 수 있습니다. 평서형 혼입이 있을 때만 최소 수정합니다.

## 승인 요청 사항

- `README.ko.md`를 `~입니다`, `~합니다`, `~하세요` 중심의 존댓말 문체로 수정하는 방향을 승인해 주세요.
- `README.zh-CN.md`는 중립 안내체 유지, `README.ja.md`는 `です/ます` 정중체 유지 기준으로 점검하고 필요 시 최소 수정하는 방향을 승인해 주세요.
- 기능/권한/개인정보/제한사항의 의미 변경은 제외하는 범위를 승인해 주세요.
