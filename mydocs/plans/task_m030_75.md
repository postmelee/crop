# Task #75 수행계획서 - Privacy Policy 적용 버전과 업데이트일 갱신

GitHub Issue: [#75](https://github.com/postmelee/crop/issues/75)
마일스톤: M030

## 목적

Chrome Web Store 제출 전 고정 Privacy URL인 `https://github.com/postmelee/crop/blob/main/PRIVACY.md`가 현재 release 기준을 명확히 나타내도록 `PRIVACY.md`의 업데이트 날짜와 적용 버전 문구를 갱신한다.

또한 이후 release마다 같은 확인을 반복할 수 있도록 `mydocs/manual/release_pipeline_guide.md`에 Privacy URL 운영 기준과 `PRIVACY.md` 날짜/적용 버전 갱신 규칙을 추가한다.

## 배경

`v0.1.1`은 `main` merge, tag, GitHub Release 생성까지 완료됐고 Chrome Web Store 제출만 남아 있다. Store Dashboard의 Privacy URL은 릴리즈마다 tag URL로 바꾸기보다 `main/PRIVACY.md` 고정 URL을 유지하는 쪽이 운영상 안정적이다.

현재 `PRIVACY.md`는 권한, 데이터 처리, local processing, no server upload, no telemetry 기준이 `v0.1.1`과 충돌하지 않는다. 다만 4개 언어 섹션의 업데이트 날짜가 `2026년 6월 6일` 기준이고, `v0.1.1 and later`에 해당하는 적용 버전 문구가 없다. 제출 전 문서 명확성을 높이고 다음 release 때 누락을 방지하려면 문서 본문과 release runbook을 함께 갱신해야 한다.

## 범위

### 포함

- `PRIVACY.md` 영어/한국어/일본어/중국어 섹션의 업데이트 날짜 갱신
- `PRIVACY.md` 4개 언어 섹션에 `crop v0.1.1 and later` 의미의 적용 버전 문구 추가
- `mydocs/manual/release_pipeline_guide.md`에 Chrome Web Store Privacy URL 운영 규칙과 반복 release 체크 기준 추가
- 단계 보고서와 최종 보고서 작성

### 제외

- `manifest.json`, `package.json` version 변경
- Chrome Web Store package 재생성
- GitHub Release `v0.1.1` tag 이동, release body 수정, release asset 교체
- Chrome Web Store Dashboard upload 또는 `Submit for review`
- Privacy policy의 데이터 처리 범위, 권한 설명, telemetry/no upload 정책 변경

## 설계 방향

- Store Dashboard에는 `https://github.com/postmelee/crop/blob/main/PRIVACY.md` 고정 URL을 우선 사용한다는 규칙을 `release_pipeline_guide.md`에 명시한다.
- tag URL은 GitHub Release note와 감사 기록용으로 유지하고, Store Dashboard URL의 기본값으로는 사용하지 않는다.
- `PRIVACY.md`의 적용 버전 문구는 각 언어 첫 설명 문단 근처에 짧게 추가한다.
- 문구는 `v0.1.1 and later, unless replaced by a newer policy` 의미를 유지하되, privacy/data handling 의미를 확장하거나 축소하지 않는다.
- 현재 release를 위해 문서만 보정하므로 package 재생성이나 release asset 교체를 하지 않는다.

### 통합 검증

- `PRIVACY.md` 4개 언어 섹션 모두 `v0.1.1` 적용 범위와 2026-06-14 업데이트 날짜를 가진다.
- `release_pipeline_guide.md`가 Chrome Web Store Privacy URL 기본값을 `main/PRIVACY.md`로 안내한다.
- `manifest.json`, `package.json`, `/tmp/crop-0.1.1-cws.zip`, GitHub Release/tag를 변경하지 않는다.
- `git status --short`가 PR 준비 전 빈 출력이다.
- `git diff --check`가 경고 없이 통과한다.

## 리스크

- **다국어 문구 불일치**: 4개 언어의 의미가 달라질 수 있다. 각 섹션에 같은 구조의 짧은 문장을 추가하고 grep으로 version/date를 확인한다.
- **Store URL 운영 혼동**: tag URL과 main URL의 역할이 섞일 수 있다. runbook에 Store Dashboard는 `main/PRIVACY.md`, GitHub Release/감사 기록은 tag URL을 우선 사용한다고 분리한다.
- **release 산출물 변경 오해**: 이 task는 문서 보정만 수행한다. package 재생성, release asset 교체, CWS 제출은 제외 범위로 유지한다.

## 승인 요청 사항

- `PRIVACY.md`에 `v0.1.1 and later` 의미의 적용 버전 문구를 4개 언어로 추가하는 범위를 승인한다.
- `Last updated`/최종 업데이트 날짜를 2026-06-14 기준으로 갱신하는 범위를 승인한다.
- `release_pipeline_guide.md`에 반복 release Privacy URL 운영 규칙을 추가하는 범위를 승인한다.

승인되면 `task_m030_75_impl.md`에서 단계별 산출물, 검증 명령, 커밋 메시지를 구체화한다.
