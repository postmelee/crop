# Task #41 수행계획서

GitHub Issue: [#41](https://github.com/postmelee/crop/issues/41)
마일스톤: M020

## 목적

Full page preview modal이 매우 긴 페이지의 캡처 결과를 단일 초대형 `<img>`로 표시하면서 빠른 스크롤 중 blank band가 보이는 문제를 해결한다. Save/Copy용 stitched PNG 생성 경로는 유지하고, modal preview 표시만 capture tile 조각 기반 renderer로 전환한다.

완료 후에는 긴 GitHub 페이지 같은 실제 장문 문서에서 full page preview를 빠르게 스크롤해도 새로 노출되는 edge에 흰색 또는 회색 blank band가 보이지 않아야 한다. visible viewport preview와 selected visible crop preview는 기존 동작을 유지한다.

## 배경

Task #40에서 preview blank band를 분석한 결과, 저장 PNG는 정상이고 문제는 초대형 단일 `<img>` preview 렌더링 artifact로 분리됐다. Stage 2의 preview image background 완화와 Stage 5의 capture wait 강화는 같은 위치의 회색 band를 남겨 근본 해결이 아니었다.

현재 full page capture는 `captureFullPageTiles()`로 viewport tile들을 수집하고 `stitchCapturedTiles()`로 Save/Copy용 PNG dataUrl을 만든다. 그러나 preview modal은 최종 stitched PNG dataUrl을 `<img class="crop-preview-image">` 한 장으로 표시한다. 이 task는 최종 PNG 생성 책임과 preview 표시 책임을 분리해, preview만 여러 작은 tile 이미지로 렌더링하는 구조를 도입한다.

## 범위

### 포함

- Full page preview mode에서 capture tile dataUrl과 `destinationCssRect` 기반 preview renderer를 설계한다.
- Save/Copy용 stitched PNG dataUrl은 기존처럼 생성하고, preview tile DOM은 저장 결과에 섞이지 않게 책임을 분리한다.
- `src/content/overlay/crop-overlay.ts`의 preview state/result model과 preview DOM 갱신 흐름을 보정한다.
- `src/content/overlay/crop-overlay.css`에 tiled preview wrapper/tile 스타일을 추가한다.
- `tests/content/overlay/phase6-regression.test.ts`, `tests/content/overlay/full-page-capture.test.ts` 등 관련 regression을 갱신한다.
- `mydocs/tech/task_m020_8_quality_matrix.md`의 P6-41을 #41 결과 기준으로 갱신한다.
- 실제 긴 GitHub 페이지 full page preview scroll 수동 smoke 기준을 최종 보고서에 남긴다.

### 제외

- Full page 저장 PNG 해상도 개선.
- multi-part PNG export, PDF export, 별도 파일 묶음 export.
- `debugger`, `<all_urls>`, broad host permission 추가.
- capture/stitching 알고리즘의 품질 변경. 단, preview tile metadata 전달에 필요한 최소 구조 변경은 포함한다.
- tile virtualization은 초기 구현에서 제외한다. 메모리나 DOM 비용이 실제 blocker로 확인되면 별도 승인 후 다룬다.

## 설계 방향

- Preview capture result는 사용자 action용 stitched PNG dataUrl과 표시용 preview model을 함께 가질 수 있게 확장한다.
- Full page mode에서만 tiled preview를 사용하고, visible viewport mode는 기존 단일 image preview를 유지한다.
- Preview tile wrapper는 stitched output의 CSS size와 같은 좌표계를 갖고, 각 tile은 `destinationCssRect`에 맞춰 absolute 배치한다.
- Downscale이 적용된 경우에도 preview wrapper/tile 좌표가 실제 표시 크기와 일관되도록 output scale/downscale metadata를 설계 단계에서 명시한다.
- Copy/Save button은 preview DOM이 아니라 기존 stitched PNG dataUrl만 사용한다.
- 권한 확대 없이 현재 `captureVisibleTab()` 기반 MV3 경계를 유지한다.

## 잠정 단계

- **Stage 1 — preview renderer contract 고정**
  - 기존 preview result, DOM, CSS, tile metadata, stitched output metadata 흐름을 분석하고 tiled preview contract를 단계 보고서에 고정한다.
  - 검증 관점: `rg`로 현재 preview/capture/stitching 경로 확인, 제품 코드 변경 없음.
- **Stage 2 — tiled preview model과 DOM 구조 구현**
  - full page preview 전용 tile model, wrapper/tile DOM 갱신, CSS 배치를 구현한다.
  - 검증 관점: typecheck, preview DOM/CSS regression, visible preview no-scroll 회귀 확인.
- **Stage 3 — Save/Copy 책임 분리와 scale 정렬 검증**
  - Copy/Save가 기존 stitched PNG dataUrl을 계속 사용하도록 보장하고, tile 좌표와 downscale metadata 정렬 테스트를 추가한다.
  - 검증 관점: full-page capture/stitch focused tests, phase6 regression.
- **Stage 4 — 통합 검증과 실제 긴 페이지 smoke**
  - build/typecheck/full test, 권한 grep, 실제 긴 GitHub 페이지 preview scroll 수동 smoke를 수행한다.
  - 검증 관점: P6-41 갱신, 저장 PNG Save smoke, 최종 보고서 작성.

### 통합 검증

- 매우 긴 실제 GitHub 페이지에서 full page preview modal을 빠르게 스크롤해도 blank band가 보이지 않는다.
- Save/Copy 결과 PNG는 기존 정상 stitched PNG와 같은 경로를 사용하고 preview tile DOM이 저장 결과에 섞이지 않는다.
- visible viewport preview는 내부 스크롤 없이 기존 맞춤 표시를 유지한다.
- full page preview toolbar, Copy/Save/Cancel/Retry 동작과 shortcut이 유지된다.
- Chrome MV3 권한이 확대되지 않는다.
- `git status --short`가 PR 준비 전 빈 출력이다.
- `git diff --check`가 경고 없이 통과한다.

## 리스크

- **메모리 사용량 증가**: stitched PNG dataUrl과 tile dataUrl을 동시에 들고 있으면 메모리 사용량이 늘 수 있다. 초기 구현에서는 정확성 우선으로 진행하되, 실제 blocker면 virtualization 또는 preview-only tile lifecycle을 별도 승인으로 다룬다.
- **tile seam/좌표 오차**: absolute tile 배치가 source/destination scale과 어긋나면 preview seam이 생길 수 있다. `destinationCssRect`, output scale, downscale metadata를 regression test로 고정한다.
- **Save/Copy 회귀**: preview DOM 변경이 action 결과에 섞이면 저장 PNG가 오염될 수 있다. Copy/Save는 stitched PNG dataUrl만 쓰도록 책임을 분리하고 기존 overlay suppression tests를 유지한다.
- **visible preview 회귀**: full page 전용 renderer가 visible mode에 영향을 줄 수 있다. visible mode는 기존 단일 image path로 유지하고 focused regression을 둔다.
- **권한 경계 확대 유혹**: Chrome compositor 문제를 `debugger`나 broad host permission으로 우회하지 않는다. 권한 grep과 manifest test를 유지한다.

## 승인 요청 사항

- Save/Copy용 stitched PNG는 유지하고 preview만 tiled renderer로 바꾸는 설계 방향
- 초기 구현에서 tile virtualization을 제외하는 기준
- full page 저장 PNG 해상도 개선, multi-part export, PDF export를 이번 task에서 제외하는 기준
- 공식 사용자 문서 신규 생성 없이 내부 품질 매트릭스와 작업 산출물만 갱신하는 기준
- 잠정 Stage 1~4 분할과 검증 계획

승인되면 `task_m020_41_impl.md`에서 단계별 산출물, 검증 명령, 커밋 메시지를 구체화한다.
