# Task #31 Stage 2 완료 보고서

GitHub Issue: [#31](https://github.com/postmelee/crop/issues/31)
구현계획서: [`task_m030_31_impl.md`](../plans/task_m030_31_impl.md)

## 단계 목적

Stage 2는 `README.ko.md`를 사용자-facing README에 맞는 존댓말 문체로 전환하고, `README.zh-CN.md`와 `README.ja.md`의 문체가 각 언어권 안내 문서 기준과 맞는지 점검하는 단계다.

이번 단계에서는 기능 설명, 권한 설명, 개인정보 설명, 제한사항의 의미를 바꾸지 않고 한국어 문장의 종결과 안내 표현만 정리했다. 중국어와 일본어 README는 현재 문체가 구현계획서 기준과 맞아 본문을 수정하지 않았다.

## 산출물

| 파일 | 변경 요약 |
|---|---|
| `README.ko.md` | 평어체 종결을 `~입니다`, `~합니다`, `~하세요` 중심의 존댓말 문체로 전환했다. |
| `README.zh-CN.md` | 수정 없음. 중국어 README에서 자연스러운 중립 안내체와 제한적 `请` 사용 상태를 확인했다. |
| `README.ja.md` | 수정 없음. 일본어 README에서 `です/ます` 정중체가 유지되는 것을 확인했다. |
| `mydocs/working/task_m030_31_stage2.md` | Stage 2 변경 내용, 검증 결과, 잔여 위험을 기록했다. |

## 본문 변경 정도 / 본문 무손실 여부

`README.ko.md`는 본문 의미를 유지한 상태에서 문체만 전환했다. 기존 기능 목록, 설치 절차, 사용법, 권한 표, 개인정보, 제한사항, 개발 섹션, 출처와 라이선스 섹션의 구조와 정보는 유지했다.

대표 변경은 다음과 같다.

- `확장이다` -> `확장입니다`
- `로드한다` -> `로드합니다`
- `연다` -> `여세요`
- `선택한다` -> `선택하세요`
- `요청하지 않는다` -> `요청하지 않습니다`
- `참고한다` -> `참고하세요`

`README.zh-CN.md`는 중국어 README 관행상 존댓말을 기계적으로 추가하지 않는 것이 자연스럽다고 판단했다. 현재 문서는 중립 안내체를 기본으로 쓰고, `请`는 요청성 문장 또는 참고 안내에서만 제한적으로 사용한다.

`README.ja.md`는 사용자 안내 문서에 적합한 `です/ます` 정중체가 이미 전반적으로 유지되어 있어 수정하지 않았다.

## 검증 결과

실행 명령:

```bash
git diff -- README.ko.md README.zh-CN.md README.ja.md
rg -n "이다\.|한다\.|된다\.|있다\.|없다\.|연다\.|선택한다\.|사용한다\.|요청한다\.|유지한다\." README.ko.md || true
rg -n "입니다|합니다|하세요|됩니다|있습니다|없습니다" README.ko.md
rg -n "请|您" README.zh-CN.md || true
rg -n "です|ます|ません|ください" README.ja.md
git diff --check
```

결과:

- OK: `git diff -- README.ko.md README.zh-CN.md README.ja.md`에서 `README.ko.md`만 변경된 것을 확인했다.
- OK: 한국어 평어체 grep은 출력이 없었다. `rg` exit code 1은 일치 항목 없음으로 해석했다.
- OK: 한국어 존댓말 grep에서 `입니다`, `합니다`, `하세요`, `됩니다`, `있습니다`, `없습니다`가 README 전반에 분포하는 것을 확인했다.
- OK: 중국어 README의 `请`는 배포 상태 안내, 권한/저장/라이선스 참고 문맥에 제한적으로 사용되고, `您`는 사용되지 않았다.
- OK: 일본어 README에서 `です`, `ます`, `ません`, `ください`가 소개, 설치, 사용법, 권한, 개인정보, 제한사항, 라이선스 문맥에 걸쳐 확인됐다.
- OK: `git diff --check`가 경고 없이 통과했다.

## 잔여 위험

- 한국어 README는 문체만 바꿨지만, 존댓말 전환 과정에서 일부 문장이 이전보다 길게 읽힐 수 있다. Stage 3에서 최종 보고 전 한 번 더 수동 검토한다.
- 중국어와 일본어 README는 본문 수정이 없으므로, 이번 task의 문체 점검 근거를 최종 보고서에 다시 요약해야 한다.
